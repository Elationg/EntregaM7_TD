Proyecto destacado en profundidad

Análisis técnico del proyecto más representativo de mi crecimiento durante la formación

⭐ Proyecto seleccionado para el portafolio

SpringEduManager — Plataforma Educativa con Spring Boot

Módulo 6 · GitHub: Elationg/EntregaM6_TD

📄 Descripción de la actividad

Desarrollo de una plataforma web educativa completa que gestiona estudiantes, cursos y evaluaciones. El sistema debía incluir autenticación segura con control de roles, una interfaz web con Thymeleaf y Bootstrap, persistencia con JPA y una API RESTful con autenticación HTTP Basic. Se integraron 5 conceptos del ecosistema Spring en un único proyecto.

⚡ Desafío principal

El mayor desafío fue integrar Spring Security 6 —con cambios significativos respecto a versiones anteriores— con las rutas de la API REST y las vistas Thymeleaf, evitando conflictos de configuración. Además, coordinar las relaciones bidireccionales entre entidades JPA sin provocar ciclos de serialización en los endpoints REST.

💡 Solución propuesta

Se configuró una clase SecurityConfig personalizada separando reglas para rutas web y API. Las relaciones JPA se manejaron con @JsonIgnore para evitar recursión infinita. Se usó H2 con persistencia en archivo para simular un entorno real sin complejidad operacional.

🔧 Herramientas técnicas utilizadas

Java 17 Spring Boot 3.2.4 Spring Security 6 Spring Data JPA Thymeleaf H2 Database Maven Bootstrap 5

🎓 Principales aprendizajes

• La importancia de configurar la seguridad como una capa explícita y separada.
• Cómo JPA maneja relaciones bidireccionales y por qué la serialización REST requiere manejo especial.
• El valor de separar responsabilidades en capas (Controller → Service → Repository) para código testeable.
• Que Spring Boot 3 requiere Java 17+ y que muchas APIs de seguridad cambiaron radicalmente respecto a 2.x.

🔑 Habilidades técnicas aplicadas

Spring MVC Spring Security 6 JPA / Hibernate REST API Design Thymeleaf Maven lifecycle Patrón MVC HTTP Basic Auth

❓ Por qué elegí este proyecto para el portafolio

Elegí SpringEduManager porque representa el punto de mayor complejidad técnica de mi formación. A diferencia de los proyectos anteriores, este requirió coordinar simultáneamente cinco capas del ecosistema Spring en un sistema coherente. Es el proyecto donde resolví problemas reales de integración (conflictos de Spring Security, relaciones JPA, serialización REST) que son exactamente los desafíos que enfrentaría en un entorno laboral real. Además, refleja mejor mi evolución: pasé de construir una aplicación de consola (M4) a una plataforma web full-stack con seguridad, persistencia y API en pocas semanas.
